# 界面模式映射说明

## 📋 界面上的三个模式按钮

根据代码分析，界面上的 **Quick**、**Standard**、**Deep** 三个按钮实际上都是**常规研究模式**，它们通过控制研究轮数来调整研究深度，而不是直接对应 QA 模式或 Benchmark 模式。

---

## 🔍 模式映射关系

### Quick 模式
- **前端设置**: `effortLevel === "quick"`
- **后端参数**: `minimum_effort=True`
- **研究轮数**: **1 轮**（固定）
- **输出格式**: 常规研究报告（不是 QA 模式）
- **特点**: 
  - 快速响应
  - 只有1轮研究循环
  - 生成完整的研究报告格式
  - 适合快速获取概览信息

**代码位置**:
```javascript
// ai-research-assistant/src/components/InitialScreen.js:464
const minimumEffort = effortLevel === "quick"
```

```python
# src/graph.py:229-234
if minimum_effort or qa_mode:
    if qa_mode:
        print("  - Using QA mode (1 loop)")
    else:
        print("  - Using minimum effort (1 loop)")
    return 1
```

---

### Standard 模式（默认）
- **前端设置**: `effortLevel === "standard"`
- **后端参数**: `extra_effort=False`, `minimum_effort=False`
- **研究轮数**: **默认轮数**（通常是 3 轮，可通过 `MAX_WEB_RESEARCH_LOOPS` 配置）
- **输出格式**: 常规研究报告
- **特点**:
  - 平衡的研究深度
  - 适中的研究轮数
  - 生成完整的研究报告
  - 适合大多数研究场景

**代码位置**:
```javascript
// ai-research-assistant/src/components/InitialScreen.js:117
const [effortLevel, setEffortLevel] = useState("standard") // 默认值
```

```python
# src/graph.py:236-249
env_max_loops = os.environ.get("MAX_WEB_RESEARCH_LOOPS")
base_max_loops = (
    int(env_max_loops)
    if env_max_loops
    else int(configurable.max_web_research_loops)
)
max_loops = base_max_loops  # 默认值，通常是 3
```

---

### Deep 模式
- **前端设置**: `effortLevel === "high"`
- **后端参数**: `extra_effort=True`
- **研究轮数**: **默认轮数**（目前代码中 `extra_effort` 不影响轮数，但可以通过配置增加）
- **输出格式**: 常规研究报告
- **特点**:
  - 更深入的研究
  - 可能执行更多研究循环（取决于配置）
  - 生成更全面的研究报告
  - 适合需要深度分析的场景

**代码位置**:
```javascript

## 📊 模式对比表

| 模式 | 前端值 | 后端参数 | 研究轮数 | 输出格式 | 工作流 |
|------|--------|---------|---------|---------|--------|
| **Quick** | `"quick"` | `minimum_effort=True` | 1 轮 | 常规报告 | `multi_agents_network` → `generate_report` → `reflect_on_report` → `finalize_report` |
| **Standard** | `"standard"` | `extra_effort=False`<br>`minimum_effort=False` | 默认（通常3轮） | 常规报告 | `multi_agents_network` → `generate_report` → `reflect_on_report` → `finalize_report` |
| **Deep** | `"high"` | `extra_effort=True` | 默认（可配置更多） | 常规报告 | `multi_agents_network` → `generate_report` → `reflect_on_report` → `finalize_report` |

## ⚠️ 重要说明

### 这三个模式都是常规模式

**关键点**:
- ✅ 这三个按钮都使用**常规研究模式**的工作流
- ✅ 它们**不直接对应** QA 模式或 Benchmark 模式
- ✅ 区别主要在于**研究轮数**，而不是工作流类型

### 与 QA 模式和 Benchmark 模式的区别

| 特性 | Quick/Standard/Deep | QA 模式 | Benchmark 模式 |
|------|-------------------|---------|---------------|
| **工作流** | `generate_report` 路径 | `generate_answer` 路径 | `generate_answer` 路径 |
| **输出格式** | 完整研究报告 | 简洁答案 | 结构化答案 |
| **引用处理** | 完整引用 | 简化引用 | 完整引用+验证 |
| **研究轮数** | Quick: 1轮<br>Standard: 3轮<br>Deep: 3+轮 | 1轮（固定） | 3-5轮（可配置） |

## 🔄 工作流对比

### Quick/Standard/Deep 模式工作流（常规模式）

```
START
  ↓
multi_agents_network (研究轮数取决于模式)
  ↓
generate_report (生成报告)
  ↓
reflect_on_report (反思报告)
  ↓
[继续研究/完成] (根据轮数和完成度决定)
  ↓
finalize_report (最终化报告)
  ↓
END
```

### QA 模式工作流

```
START
  ↓
multi_agents_network (1轮)
  ↓
validate_context_sufficiency
  ↓
generate_answer (生成答案)
  ↓
reflect_answer (反思答案)
  ↓
finalize_answer (最终化答案)
  ↓
END
```

### Benchmark 模式工作流

```
START
  ↓
multi_agents_network (多轮)
  ↓
validate_context_sufficiency
  ↓
generate_answer (生成答案)
  ↓
reflect_answer (反思答案)
  ↓
[继续研究/完成]
  ↓
finalize_answer (最终化答案)
  ↓
[可选] verify_answer (验证答案)
  ↓
END
```

## 💡 使用建议

### 何时使用 Quick 模式？
- ✅ 需要快速获取信息概览
- ✅ 简单的研究主题
- ✅ 时间有限
- ✅ 不需要深入分析

### 何时使用 Standard 模式？
- ✅ 大多数常规研究场景
- ✅ 平衡质量和速度
- ✅ 一般深度的研究需求

### 何时使用 Deep 模式？
- ✅ 需要深入分析
- ✅ 复杂的研究主题
- ✅ 需要全面的信息覆盖
- ✅ 有充足的时间

# 第209-249行：获取最大研究轮数
def get_max_loops(configurable, extra_effort=False, minimum_effort=False, 
                  benchmark_mode=False, qa_mode=False):
    # Quick 模式：minimum_effort=True → 返回 1 轮
    if minimum_effort or qa_mode:
        return 1
    
    # Standard/Deep 模式：使用默认轮数
    base_max_loops = configurable.max_web_research_loops  # 默认 3
    return base_max_loops
```

## 📝 总结

1. **Quick、Standard、Deep 都是常规研究模式**，区别在于研究轮数
2. **Quick** = 1轮研究（`minimum_effort=True`）
3. **Standard** = 默认轮数（通常3轮）
4. **Deep** = 默认轮数（`extra_effort=True`，可通过配置增加）
5. **QA 模式和 Benchmark 模式**需要通过其他方式设置（如 `mode === "benchmark"`）

这三个按钮提供了**研究深度的选择**，而不是**研究模式的切换**。
